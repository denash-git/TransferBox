# TransferBox

TransferBox — минималистичная TUI-утилита управления прокси-сервером для Debian 13 на базе связки **Caddy** и **sing-box**.

Проект объединяет несколько протоколов на одном порту **443** (HTTPS) с маскировкой под полноценный веб-сайт (Fake Site) для обхода блокировок.

---

## Возможности

*   **Все на порту 443**: Трафик распределяется на уровне фронтенда:
    *   **NaiveProxy**: Трафик дешифруется с вашим SSL-сертификатом и пересылается в локальный SOCKS5-inbound `sing-box`.
    *   **VLESS (WebSocket/gRPC)**: Запросы по секретному пути перенаправляются на VLESS-inbound `sing-box`.
    *   **Обычные посетители и сканеры**: Видят полноценный фейковый сайт-заглушку.
*   **Автоматический SSL**: Caddy сам запрашивает и продлевает бесплатные TLS-сертификаты от Let's Encrypt для вашего домена.
*   **Удобное меню псевдографики (TUI)**: Управление пользователями, просмотр ссылок подключения и QR-кодов прямо в консоли.
*   **Минимализм**: Никаких панелей, баз данных (только простой JSON-файл) и лишнего софта.

---

## Требования

*   Debian 13 (или 12)
*   Root-доступ на сервере
*   Свободные порты `80` и `443`
*   Доменное имя, настроенное (DNS A-запись) на IP вашего сервера

## Установка

Запустите установщик одной командой (замените `ВАШ_GITHUB_АККАУНТ` на имя вашего профиля):

```bash
bash <(curl -fsSL "https://raw.githubusercontent.com/ВАШ_GITHUB_АККАУНТ/TransferBox/main/install.sh")
```

---

## Управление

После завершения установки откройте консольное меню управления командой:

```bash
transferbox
```

В меню доступны:
1. Просмотр списка пользователей, их ссылок подключения и QR-кодов.
2. Добавление новых пользователей (NaiveProxy / VLESS WS / VLESS gRPC).
3. Включение / выключение учетных записей.
4. Удаление пользователей.
5. Настройки домена, email и маскировки.
6. Перезапуск прокси-служб и чтение логов.
