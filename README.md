**TransferBox** — это современная, легковесная и высокобезопасная панель управления прокси-сервером на базе связки **Caddy** и **sing-box** с продвинутой маскировкой под веб-сайт и защитой от активного сканирования (Active Probing). 

Все протоколы работают на едином веб-порту **443** (HTTPS), а нелегитимные запросы бесшовно перенаправляются на маскировочный сайт с одним из нескольких футуристических шаблонов дизайна.

---

## ✨ Основные Возможности

### 🔒 Поддерживаемые Протоколы и Модули
*   **VLESS over WebSocket**: Стабильный протокол, совместимый с любыми CDN.
*   **VLESS over gRPC**: Быстрый протокол с низким оверхедом заголовков.
*   **VLESS over XHTTP**: Новейший HTTP-транспорт sing-box для обхода глубокого анализа пакетов (DPI).
*   **NaiveProxy**: Сверхнадежная маскировка трафика под стандартный HTTP/2 благодаря движку Chromium.
*   **Mieru (mita)** *(опционально)*: Высокоустойчивый к блокировкам SOCKS5-прокси на базе технологии шифрования с имитацией шума.
*   **NetBird** *(опционально)*: Бесшовная интеграция с приватной Mesh-VPN сетью на базе WireGuard для объединения ваших серверов.

### 🛠️ Интерактивный TUI (Терминальный интерфейс)
*   **Pure Bash Navigation**: Управление с помощью стрелок (`↑↓`), клавиш `Enter`, `q` (назад) и `0` (выход) без необходимости мыши.
*   **Zero-Fork рендеринг**: Высокая производительность и отсутствие задержек (информация о состоянии служб кэшируется в оперативной памяти).
*   **Мониторинг ресурсов**: Реализован встроенный экран мониторинга VPS в реальном времени (нагрузка CPU, использование RAM/Swap, дисковое пространство, сетевой трафик RX/TX и системный аптайм).
*   **Умное логирование**: Меню просмотра логов Caddy, sing-box, а также подключенных модулей Mieru и NetBird глубиной 100 строк (с возможностью следить за логом в реальном времени через `journalctl -f`).

### 🛡️ Безопасность и Архитектура
*   **Жесткие права доступа (ACL)**: Конфигурационные файлы (пароли NaiveProxy, ключи шифрования) сохраняются с правами `0600` / `0640`. Папка подписок `/opt/transferbox/sub` защищена правами `0700`.
*   **Изолированные службы**: Службы запущены от системных пользователей с минимальными правами.
*   **Строгий UFW**: Автоматическая настройка фаервола. По умолчанию открыты только порты SSH, 80/tcp, 443/tcp и 443/udp (для HTTP/3). Все остальные порты закрыты.
*   **Временные файлы (`mktemp`)**: Полная защита от атак типа Race Condition и Symlink благодаря динамической генерации временных путей.

## 🚀 Быстрый Старт

### Системные требования
*   **ОС:** Чистая Debian 12 / 13 или Ubuntu 22.04 / 24.04 LTS (с правами root).
*   **Домен:** А-запись домена должна быть заранее направлена на публичный IP-адрес вашего сервера.
*   **Порты:** Порты `80` и `443` должны быть свободны.

### Установка (в одну команду)
Просто запустите установщик в консоли вашего сервера:
```bash
bash <(wget -qO- https://raw.githubusercontent.com/denash-git/TransferBox/main/install.sh)
```
*Установщик автоматически скачает необходимые пакеты, настроит демоны, сгенерирует сертификаты Let's Encrypt и подготовит среду.*

### Управление панелью
Запустите интерактивный терминальный интерфейс:
```bash
menu
```

## 🎛️ Структура Панели Управления

*   **👥 Пользователи**
    *   *Просмотр списка*: Отображение всех пользователей, их протоколов, статуса активности и персональных ссылок подписки.
    *   *Создание пользователя*: Мастер генерации пользователя (VLESS WS / gRPC / XHTTP, NaiveProxy, Mieru) с автоматическим выпуском UUID и паролей.
    *   *Добавление протокола*: Возможность привязать несколько различных протоколов к одному никнейму.
    *   *Управление доступом*: Мгновенное включение / выключение (блокировка) пользователя и его полное удаление.
    *   *QR-код и подписка*: Генерация QR-кодов для импорта в клиенты (v2rayN, Shadowrocket, Nekobox, Sing-Box) и ссылок на автообновляемые подписки.
*   **⚙️ Настройки**
    *   *Домен и Email*: Быстрая смена домена или адреса почты Let's Encrypt.
    *   *VPS FQDN*: Смена внутреннего имени сервера.
    *   *Шаблон маскировочного сайта*: Переключение тем сайта-заглушки (на выбор 5 футуристических и строгих дизайнов сайтов).
    *   *Пути VLESS*: Перегенерация секретных путей WebSocket/XHTTP и имени службы gRPC.
    *   *Порт SSH*: Безопасная смена системного порта SSH с автонастройкой фаервола.
    *   *TCP BBR*: Переключение оптимизации перегрузки сети Google BBR.
    *   *Проверить обновления панели*: Инструмент проверки версий и обновления в один клик.
*   **🧩 Модули**
    *   *Mieru*: Включение/выключение, смена входящего порта.
    *   *NetBird*: Привязка сервера к приватной сети NetBird по Setup Key.
*   **🔍 Диагностика**
    *   *Пересоздать конфигурации*: Полная регенерация всех файлов конфигураций и перезапуск демонов.
    *   *Показать логи*: Просмотр журналов системных служб.
    *   *Мониторинг ресурсов*: Терминальный дашборд использования ресурсов VPS.
    *   *Тест скорости (Client ↔ VPS)*: Временный запуск iperf3-сервера с автоматическим открытием портов в UFW и подробной инструкцией по тестированию пропускной способности.
*   **💾 Бэкап и Восстановление**
    *   Создание резервных копий конфигураций и базы данных в `/opt/transferbox/backups/`.
    *   Откат настроек из имеющихся резервных копий.

## 🔄 Система Обновлений и Откатов (Transactions)

Начиная с версии 2.0.0, в TransferBox встроен **транзакционный автоапдейтер** с защитой от сбоев:

1.  **Проверка версий**: Выполняется раз в час в фоне (асинхронно). Если найдено обновление, в шапке панели загорается зеленый бейдж `[↑ update vX.X.X]`.
2.  **Загрузка релизов по тегам**: Архив скачивается в формате `tar.gz` конкретного стабильного релиза с GitHub, исключая установку промежуточного кода из ветки `main`.
3.  **Резервное копирование**: Перед применением обновления создаются бэкапы всей рабочей папки `core/`, базы пользователей `users.json` и настроек `instance.env`.
4.  **Валидация синтаксиса**: Скрипты автоматически тестируются на ошибки синтаксиса (`bash -n` и `py_compile`).
5.  **Тестирование запуска служб**: Новые конфигурации применяются и валидируются. Если службы не запустились, или упала база данных — **автоапдейтер совершает полный откат** кода и данных до стабильного состояния.
6.  **Миграции БД (`run_migrations`)**: Поддерживается версионирование схемы БД через параметр `SCHEMA_VERSION` в `instance.env`. Обновления базы (например, при переходе с v1 на v2) накатятся автоматически без потери данных пользователей.
