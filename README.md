# 📦 TransferBox v3.0.1

[![OS Support](https://img.shields.io/badge/OS-Debian%2012%20%7C%2013%20%7C%20Ubuntu%2022.04%20%7C%2024.04-blue.svg)](https://debian.org)
[![Version](https://img.shields.io/badge/version-3.0.1-green.svg)](#)

**TransferBox** — это универсальная и удобная панель управления прокси и VPN-соединениями с маскировкой трафика под обычный веб-сайт. Она объединяет передовые протоколы и инструменты безопасной связи на едином HTTPS-порту.

---

## ✨ Возможности панели

*   **Поддерживаемые протоколы и модули:**
    *   VLESS (WebSocket, gRPC, XHTTP)
    *   NaiveProxy
    *   Mieru (mita)
    *   NetBird
*   **Интерактивный TUI-интерфейс:** Удобное меню управления пользователями, протоколами и подписками в терминале (управление стрелками клавиатуры).
*   **Интеграция с Telegram-ботом:** Управление пользователями, просмотр статуса сервера, генерация ссылок подключения и получение мгновенных уведомлений (алертов) о состоянии системы прямо в Telegram.
*   **Встроенная диагностика и мониторинг:** Быстрый просмотр системных логов, мониторинг ресурсов сервера в реальном времени и тест скорости.
*   **Резервное копирование:** Создание зашифрованных бэкапов и восстановление настроек в один клик.
*   **Безопасное обновление:** Проверка и обновление панели напрямую из меню настроек с автоматической защитой от сбоев при установке.

---

## 💻 Требования

*   **ОС:** Чистая Debian 12 / 13 или Ubuntu 22.04 / 24.04 LTS с root-доступом
*   **Домен:** Доменное имя должно быть направлено (DNS A-запись) на IP-адрес вашего сервера
*   **Порты:** Свободные порты `80` и `443`

## 🚀 Установка и запуск

### 1. Установка на сервере
Выполните команду в консоли вашего сервера:
```bash
bash <(wget -qO- https://raw.githubusercontent.com/denash-git/TransferBox/main/install.sh)
```

### 2. Запуск меню управления
После успешного завершения установки запустите интерфейс управления командой:
```bash
menu
```

## 🛠️ Руководство по настройке и поддержке

### 1. Важные нюансы при первой установке и DNS
*   **Направление домена:** Перед запуском установки обязательно убедитесь, что DNS A-запись вашего домена указывает на IP-адрес сервера. Доменная запись должна быть направлена до начала процесса установки для успешного выпуска SSL-сертификата.

### 2. Резервное копирование и перенос на другой сервер
Вы можете легко перенести все настройки панели, пользователей и конфигурации на другой сервер:
1.  **Создание бэкапа**: Перейдите в `menu` -> `Настройки` -> `Бэкап и Восстановление` -> `Создать бэкап настроек`.
2.  **Защита паролем**: Вы можете защитить архив паролем. Для этого в том же меню выберите `Задать пароль для бэкапов` и укажите сложную комбинацию.
3.  **Перенос**: 
    * Установите панель на новый сервер через стандартный скрипт установки.
    * Перенесите сгенерированный zip-файл бэкапа со старого сервера на новый в директорию `/opt/transferbox/backups/`.
    * Запустите на новом сервере `menu` -> `Настройки` -> `Бэкап и Восстановление` -> `Восстановить из последнего бэкапа`.
    * Если бэкап был зашифрован, TUI затребует пароль. Если пароль совпадает с текущим из `instance.env` нового сервера, расшифровка произойдет автоматически.
    * По окончании восстановления все службы (включая Telegram-бота, если он был активен) запустятся автоматически.

### 3. Интеграция с Telegram и алерты
Связка с Telegram-ботом позволяет получать оперативные уведомления о здоровье сервера (нагрузка CPU, состояние RAM/Disk, статус системных служб) и высылать ссылки пользователям.
*   **Включение бота**: В меню бота введите токен и Chat ID. Бот автоматически создаст системную службу и запустит демон.
*   **Безопасность пароля бэкапов**: В целях безопасности управление паролем шифрования осуществляется только через консольный TUI. В Telegram-боте отображается исключительно статус активности пароля (задан / не задан), что исключает компрометацию учетных данных при чтении переписки.

### 4. Особенности подписок и импорта конфигураций ⚠️

Для каждого пользователя генерируется универсальная ссылка подписки. При её обновлении в клиентском приложении конфигурации VLESS-протоколов подтягиваются автоматически.

**Важно:** протоколы **NaiveProxy** и **Mieru** присутствуют в файле подписки, однако не все клиентские приложения их распознают и добавляют автоматически. Если после обновления подписки эти протоколы не появились — добавьте их вручную:

*   **Через Telegram-бота**: откройте карточку нужного пользователя → выберите протокол → нажмите **«🔑 Получить ссылку & QR»**. Бот пришлёт ссылку подключения и QR-код, а для Mieru — дополнительно Sing-box JSON конфигурацию с QR-кодом.
*   **Через консольное меню**: `menu` → `Пользователи` → выберите пользователя → выберите протокол → `Показать данные подключения`.
